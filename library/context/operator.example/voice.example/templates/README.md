# Structural templates

Content-stripped blueprints of the deliverable shapes that work for you — the skeleton, not the content. Fill with the actual argument when drafting; don't deviate from a proven shape without a reason.

Add one file per recurring deliverable type (e.g. `carousel-arc.md`, `linkedin-body.md`). Each holds the structural sequence, not example prose. Build these from deliverables that shipped well.
